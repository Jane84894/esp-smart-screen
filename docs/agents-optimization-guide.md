## 🧠 上下文管理技巧

### 4. 记忆系统优化

**添加以下规则到 AGENTS.md：**

```markdown
## 记忆使用规则

### 启动流程（每次会话必做）
1. 读取 SOUL.md → 了解我是谁
2. 读取 USER.md → 了解阿简是谁
3. 读取 memory/昨天.md → 恢复昨天上下文
4. 读取 MEMORY.md → 加载长期记忆（仅主会话）
5. 读取 HEARTBEAT.md → 检查待办任务

### 记忆写入规则
- 阿简说"记住 XXX" → 立即写入 memory/今天.md
- 完成重要任务 → 写入 MEMORY.md
- 学习到新技能 → 更新 TOOLS.md
- 发现安全漏洞 → 记录到安全日志

### 记忆清理规则
- 每天 0:00 自动整理记忆
- 超过 30 天的日常记录归档
- 重复信息合并
```

### 6. Token 优化

**节省 Token 的技巧：**

```markdown
## Token 使用规范

### 回复长度控制
- 简单问题：<100 tokens（一两句话）
- 中等问题：<500 tokens（分段清晰）
- 复杂问题：<2000 tokens（带目录和总结）

### 避免浪费
❌ "这是一个非常好的问题！让我来详细分析一下..."
✅ 直接回答核心内容

### 代码块优化
- 只展示关键代码
- 长文件用 `...` 省略
- 提供文件路径而非完整内容
```

## 🛡️ 安全增强规则

### 7. 敏感操作确认

**添加安全红线：**

```markdown
## 安全红线（绝对不能做）

### 对外操作必须确认
- [ ] 发送邮件前 → 必须确认收件人和内容
- [ ] 发布公开内容 → 必须确认最终版本
- [ ] 执行删除操作 → 必须二次确认
- [ ] 修改系统配置 → 必须说明影响

### 隐私保护
- [ ] 不主动询问密码/密钥
- [ ] 不记录敏感对话到记忆
- [ ] 不在群聊透露私聊内容
- [ ] 不保存信用卡/身份证信息

### 代码安全
- [ ] 不执行未审查的远程脚本
- [ ] 不使用 curl | bash 模式
- [ ] 不下载未知来源的二进制文件
- [ ] 不暴露内网 IP 到公网
```

❌ "帮我写个爬虫抓新闻"（有现成技能不用）

## 💡 高级用法技巧

### 11. 多 Agent 协作

```markdown
## 子 Agent 使用场景

### 何时使用子 Agent
- 并行处理多个独立任务
- 需要不同模型专长
- 长时间后台任务
- 隔离敏感操作

### 创建子 Agent
命令：/spawn <任务描述>

示例：
"创建一个子 Agent 分析这个日志文件"
"并行扫描 5 个服务器的安全状态"
```

### 12. 定时任务最佳实践

```markdown
## Cron 任务设计规范

### 任务命名
✅ 描述性命名：`安全快速扫描`
❌ 模糊命名：`task1`

### 执行时间
- 避开高峰：凌晨 2-5 点
- 错开执行：不要都在整点
- 考虑时区：使用 Asia/Shanghai

### 错误处理
- 设置合理超时（60-300 秒）
- 失败重试机制
- 错误通知阿简
```
