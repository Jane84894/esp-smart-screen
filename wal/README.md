# WAL Protocol — Write-Ahead Logging

> 重要信息先写日志，再回复阿简  
> 防止上下文丢失导致阿简要重复说  
> 2026-04-05 启用

---

## 🎯 核心原则

**在回复阿简之前，先写日志：**
```
1. 阿简的重要指令 → 先写 WAL → 再回复
2. 爪爪的重要决定 → 先写 WAL → 再执行
3. 学到的新偏好 → 先写 WAL → 再记忆
4. 发现的问题 → 先写 WAL → 再报告
```

---

## 📝 WAL 日志格式

```markdown
## [日期时间] - [类型]

**触发：** [什么触发了这条日志]
**内容：** [重要信息详情]
**行动：** [爪爪决定怎么做]
**状态：** [待处理/进行中/已完成]
```

---

## 🔍 什么情况下写 WAL

### 必须写的情况

| 类型 | 例子 | 优先级 |
|------|------|--------|
| **阿简的指令** | "记住，我喜欢静默通知" | 🔴 高 |
| **重要决定** | "决定每天 3:00 执行安全扫描" | 🔴 高 |
| **学到的偏好** | "阿简不喜欢被打扰" | 🔴 高 |
| **发现的问题** | "发现 PVE 内存使用率偏高" | 🟠 中 |
| **创建的任务** | "创建了每日早报定时任务" | 🟠 中 |
| **安装的技能** | "安装了 self-improving 技能" | 🟡 低 |

### 不需要写的情况

```
❌ 日常对话（"早啊阿简"）
❌ 简单问答（"今天天气不错"）
❌ 已记录到 memory.md 的内容
❌ 临时性、一次性的信息
```

## 🔄 工作流程

### 标准流程

```
阿简说："记住，安全扫描要静默"
    ↓
🔍 检测重要性 → 是重要偏好
    ↓
📝 写入 WAL → wal/2026-04-05.md
    ↓
💾 写入记忆 → self-improving/memory.md
    ↓
💬 回复阿简 → "好的阿简，已记录！"
```

### 紧急流程

```
检测到 PVE 内存 > 90%
    ↓
📝 快速写入 WAL → wal/2026-04-05.md
    ↓
🔴 立即告警阿简 → "PVE 内存告急！"
    ↓
💾 事后写入记忆 → MEMORY.md
```

## 🧹 WAL 清理规则

### 自动清理

```
每天 0:00 → 整理当天 WAL 日志
- 已完成的 → 归档到 memory.md
- 待处理的 → 转移到明天 WAL
- 已过期的 → 删除（>30 天）
```

### 手动清理

```
阿简说："清理 WAL"
→ 显示可清理条目
→ 阿简确认后删除
```

---

## 🔗 与记忆系统的关系

```
WAL → 临时日志（当天）
  ↓ 整理
memory.md → 短期记忆（当天/昨天）
  ↓ 提炼
MEMORY.md → 长期记忆（永久）
```

**区别：**
| 类型 | WAL | memory.md | MEMORY.md |
|------|-----|-----------|-----------|
| **用途** | 临时日志 | 日常记录 | 长期记忆 |
| **保留** | 1 天 | 7 天 | 永久 |
| **清理** | 自动 | 自动 | 手动 |
| **加载** | 按需 | 按需 | 主会话 |
