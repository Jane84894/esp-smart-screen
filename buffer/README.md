# Working Buffer Protocol

> 对话缓冲区 — 防止上下文丢失  
> 2026-04-05 启用

---

## 🎯 核心功能

**记录每次对话到缓冲区，上下文丢失后可恢复：**
```
1. 阿简的问题 → 记录到 buffer
2. 爪爪的回答 → 记录到 buffer
3. 上下文丢失 → 从 buffer 恢复
4. 长对话 → 自动分段保存
```

---

## 📁 Buffer 文件结构

```
~/.openclaw/workspace/buffer/
├── current.md → 当前对话缓冲
├── archive/ → 归档的对话
└── README.md → 使用说明
```

---

## 🔄 工作流程

### 对话记录

```
阿简："帮我扫描内网"
    ↓
📝 记录到 buffer/current.md
    ↓
🦞 爪爪执行扫描
    ↓
📝 记录结果到 buffer/current.md
    ↓
💬 回复阿简
```

### 上下文恢复

```
检测到上下文丢失
    ↓
📂 读取 buffer/current.md
    ↓
🧠 恢复对话历史
    ↓
💬 告诉阿简："爪爪刚才断线了，让我看看刚才说到哪..."
```

---

## 📊 Buffer 格式

```markdown
## [日期时间] - [轮次]

**阿简：** [问题/指令]
**爪爪：** [回答/行动]
**状态：** [进行中/已完成]
**上下文：** [相关背景]
```

---

## 🧹 清理规则

```
每天 0:00 → 整理 buffer
- 已完成的对话 → 归档到 archive/
- 进行中的对话 → 保留在 current.md
- 超过 7 天的归档 → 删除
```
